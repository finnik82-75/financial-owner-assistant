# Financial Owner Assistant

ИИ-ассистент для анализа управленческой отчётности собственника бизнеса. Пользователь загружает Excel или PDF с БДР и ДДС и получает структурированный управленческий отчёт на понятном языке без глубоких финансовых знаний.

## Что умеет MVP

- Принимает файлы Excel (`.xlsx`, `.xls`) и PDF с отчётами БДР и ДДС.
- Автоматически определяет отчётный период: месяц, квартал или год.
- Автоматически распознаёт проекты и направления деятельности.
- Рассчитывает управленческие KPI: выручка, операционная прибыль, маржа, точка безубыточности, кассовый разрыв.
- Строит сравнительный анализ рентабельности по проектам.
- Генерирует владельческий отчёт в свободном тексте с выводами, рисками и рекомендациями.
- Отвечает на вопросы по отчёту в режиме Q&A (экспериментальный модуль).

## Ключевая идея

> Python считает, LLM объясняет.

Все цифры и KPI рассчитываются детерминированно в Python. LLM используется для интерпретации уже готового контекста и формирования понятного управленческого текста.

## Архитектура пайплайна

```text
Загрузка файла
    │
    ▼
Парсинг (Excel / PDF)
    │
    ▼
Нормализация данных
    │
    ▼
Расчёт KPI
    │
    ▼
Построение контекста анализа
    │
    ▼
Генерация владельческого отчёта
    │
    ▼
Q&A по отчёту (экспериментальный модуль)
```

### Q&A: трёхзвенная архитектура

1. **Плановик** (`analytical_query_planner.py`) — классифицирует запрос и строит план обработки на основе правил; LLM используется только как запасной вариант.
2. **Исполнитель** (`analytical_executor.py`) — выполняет детерминированные Python-обработчики; LLM не видит сырые числа.
3. **Компоновщик** (`answer_composer.py`) — LLM объясняет уже готовый payload простым человеческим языком.

## Быстрый старт

```bash
# 1. Создать виртуальное окружение
python -m venv venv

# 2. Активировать (Windows)
venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Скопировать и заполнить конфигурацию
copy .env.example .env

# 5. Запустить сервер
uvicorn app.main:app --reload --port 8010
```

Приложение будет доступно по адресу [http://localhost:8010](http://localhost:8010).

## Переменные окружения

Минимально необходимые переменные в файле `.env`:

| Переменная | Назначение | Пример |
| --- | --- | --- |
| `OPENAI_API_KEY` | Ключ OpenAI API | `sk-...` |
| `OPENAI_MODEL` | Модель по умолчанию | `gpt-4o-mini` |
| `OPENAI_TEMPERATURE` | Температура генерации | `0.2` |

Дополнительные переменные:

| Переменная | Назначение | Дефолт |
| --- | --- | --- |
| `OPENAI_MODEL_FAST` | Быстрая модель для планировщика | `gpt-4o-mini` |
| `OPENAI_MODEL_STRONG` | Более сильная модель для отчёта | `gpt-4o` |
| `MAX_FILES` | Максимум файлов за сессию | `10` |
| `MAX_FILE_SIZE_MB` | Максимум размера одного файла | `20` |
| `MAX_TOTAL_SIZE_MB` | Максимум суммарного размера | `100` |

Если `OPENAI_API_KEY` отсутствует или недействителен, владельческий отчёт и Q&A не работают. Парсинг, нормализация и расчёт KPI продолжают работать.

## Загрузка отчётов

| Метод | URL | Описание |
| --- | --- | --- |
| GET | `/upload` | Страница загрузки |
| POST | `/upload` | Загрузить один или несколько файлов |
| POST | `/extract` | Распарсить загруженные файлы |
| POST | `/normalize` | Нормализовать данные |
| GET | `/report/{report_id}` | Просмотр владельческого отчёта |
| GET | `/kb` | Статус базы знаний |
| POST | `/ask-page` | Страница Q&A с ответом (HTML) |
| POST | `/ask` | Q&A API (JSON) |

Поддерживаемые форматы: `.xlsx`, `.xls`, `.pdf`.
Можно загружать несколько файлов одновременно, например БДР и ДДС отдельными файлами.

## Структура результата

После нормализации формируется JSON-контекст со следующими блоками:

```json
{
  "period": {
    "period_label": "за 1 квартал 2026 года",
    "months": []
  },
  "pnl": {
    "totals": { "revenue": 0, "operating_profit": 0 },
    "monthly": { "2026-01": {} },
    "projects": { "Европа+": {} }
  },
  "cashflow": {
    "totals": { "operating_cashflow": 0, "net_cashflow": 0 },
    "details": { "payroll": 0, "contractors": 0 }
  },
  "kpi": {
    "summary_kpi": { "gross_margin": 0, "break_even_revenue": 0 },
    "monthly_kpi": {},
    "project_kpi": {}
  }
}
```

На этот контекст опираются и генератор отчёта, и Q&A-пайплайн.

## Ограничения MVP

1. **Зависимость от качества файлов.** Парсер читает структуру таблиц Excel и текст PDF. Нестандартные форматы, объединённые ячейки или сканированные PDF могут давать неполный результат.
2. **Прибыль проекта — proxy-метрика.** Расходы распределяются пропорционально выручке. Точные затраты по проектам требуют раздельного учёта в исходном отчёте.
3. **Чистая прибыль = операционная прибыль.** Налоги на прибыль, проценты по кредитам и прочие внеоперационные статьи в MVP не выделяются отдельно.
4. **Q&A — экспериментальный модуль.** Охватывает базовые вопросы по расходам, проектам, трендам и KPI. Нестандартные многошаговые вопросы могут дать неточный или неполный ответ.
5. **Сложные сценарии требуют ручной проверки.** MVP не делает план-фактный анализ, не строит прогнозы и не работает с несколькими периодами одновременно.
6. **LLM не является источником расчётов.** Все числа рассчитываются детерминированным Python-кодом. LLM только объясняет заранее вычисленные значения.

5. Автоматическое определение проектов.
6. Расчёт KPI (маржа, точка безубыточности).
7. Генерация владельческого отчёта.
8. Q&A: простой вопрос по расходам.
9. Q&A: вопрос по убыточным и прибыльным проектам.
10. Корректная работа без ключа OpenAI.

## Что не хранится в репозитории

- `venv/` — виртуальное окружение Python
- `.env` — секреты и конфигурация
- `data/uploads/`, `data/parsed/`, `data/outputs/` — пользовательские данные и отчёты

Для сохранения структуры в Git используются `.gitkeep`-файлы в соответствующих директориях.
